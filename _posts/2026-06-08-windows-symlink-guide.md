## 一、你遇到了什么痛点？

你有没有遇到过这些让人血压飙升的情况：

- 某个软件**死活不让改安装路径**，强行往 C 盘塞
- VS Code 更新时**只能装在 C 盘**，旧版本明明在 D 盘
- Steam 游戏库越来越大，C 盘红了但**不想重装游戏**
- 某些应用的缓存/数据目录**默认在 C 盘且改不了**

**用"符号链接"（Symbolic Link）可以一招解决以上所有问题。**

## 二、什么是符号链接？用人话解释

> 在 C 盘放一个"遥控器"，按下按钮，实际干活的是 D 盘里的真家伙。系统和软件都以为东西还在 C 盘，其实早就搬走了。

**打个比方：**

你搬家了，但怕朋友找不到，于是在旧家门口贴了张纸条："我搬到 XXX 了，以后来这儿找我"。朋友走到旧家，看到条，顺着地址找到新家。

- **C 盘的符号链接** = 旧家门口的纸条（一个"快捷方式"）
- **D 盘的实际文件夹** = 你的新家（真实数据）
- **系统 / 软件** = 你的朋友（被完美地"骗"了过去）

核心要点：

| 概念 | 说明 |
|------|------|
| 符号链接本身 | 在 C 盘，只是个"指路牌"，几乎不占空间 |
| 实际数据 | 在 D 盘，所有文件真正存在这里 |
| 对软件的影响 | 完全透明，软件感知不到任何变化 |

## 三、创建符号链接：超详细步骤

### 3.1 准备工作

假设场景：
- 顽固软件把数据存在 `C:\StubbornData`
- 你想把它实际放到 `D:\MyRealData`

### 3.2 操作步骤

**第一步：移动文件夹**

把 `C:\StubbornData` 整个文件夹**剪切**，粘贴到 D 盘目标位置 `D:\MyRealData`。

> ⚠️ 关键：移动成功后，C 盘原来的 `C:\StubbornData` 必须已经不存在了！

**第二步：以管理员身份打开命令行**

1. 按 `Win + S`，搜索 `cmd` 或 `命令提示符`
2. 在搜索结果中，**右键** → **以管理员身份运行**
3. 这一步极其重要！没有管理员权限，以下命令会失败

**第三步：执行命令**

```cmd
mklink /J "C:\StubbornData" "D:\MyRealData"
```

**命令拆解**：

| 部分 | 含义 |
|------|------|
| `mklink` | Windows 创建链接的命令 |
| `/J` | 创建"目录联接"（Junction），兼容性最好，推荐使用 |
| `"C:\StubbornData"` | C 盘原位置（此时这个文件夹已不存在） |
| `"D:\MyRealData"` | D 盘真实数据所在位置 |

**第四步：确认成功**

如果看到这行提示，就说明成功了：

```
为 C:\StubbornData <<===>> D:\MyRealData 创建的联接
```

此时回 C 盘看，`C:\StubbornData` 文件夹又重新出现了，图标的左下角可能有一个小小的**快捷方式箭头**——这就是符号链接的标记。

## 五、如何正确删除符号链接？

**这是最容易踩坑的地方，请一定仔细看！**

### 核心原则

```
只删 C 盘的"假文件夹"（符号链接），别动 D 盘的"真数据"
```

### 正确操作

**方法一：直接右键删除（推荐）**

1. 去 C 盘找到那个带小箭头图标的符号链接文件夹
2. 右键 → 删除（或按 `Delete` 键）
3. 确认删除

> ✅ 这个操作**只删除链接本身**，D 盘真实数据毫发无伤。

**方法二：命令行删除**

以管理员身份运行 CMD，执行：

```cmd
rmdir "C:\StubbornData"
```

### ⚠️ 千万不能这样做！

```
❌ 绝对不要先去 D 盘删除或移动实际文件夹！
```

如果先删了 D 盘的真实数据：

1. C 盘的符号链接还在
2. 但它指向的目标已经不存在了
3. 它变成了**"断开的链接"（幽灵文件夹）**
4. 无法打开、可能无法正常删除，非常麻烦

### 如果已经搞砸了怎么办？

如果手快先删了 D 盘文件，导致 C 盘出现"幽灵文件夹"，补救方法：

```cmd
# 以管理员身份运行 CMD
rmdir "C:\StubbornData"
```

这个命令可以强制删除失效的符号链接。

### 正确顺序总结

```
想取消符号链接？
  → 第 1 步：删 C 盘（带小箭头的假文件夹）
  → 第 2 步：再决定 D 盘真实数据要不要删
```

## 八、命令速查表

| 操作 | 命令 |
|------|------|
| 创建目录联接 | `mklink /J "C盘路径" "D盘路径"` |
| 创建符号链接（文件） | `mklink "C盘路径" "D盘路径"` |
| 创建硬链接 | `mklink /H "C盘路径" "D盘路径"` |
| 强制删除失效链接 | `rmdir "失效链接路径"` |

### `/J` 参数说明

| 参数 | 类型 | 适用场景 |
|------|------|----------|
| `/J` | 目录联接（Junction） | **推荐**，兼容性最好，适合文件夹 |
| `/D` | 目录符号链接 | 需要管理员权限，支持网络路径 |
| `/H` | 硬链接 | 仅适用于文件，不能跨盘 |
| 无参数 | 文件符号链接 | 适用于单个文件 |

> 💡 对于"把 C 盘文件夹搬到 D 盘"这个场景，一律用 `/J`（目录联接）就对了。

## 九、常见问题 FAQ

### Q1：符号链接和快捷方式（.lnk）有什么区别？

| | 快捷方式 (.lnk) | 符号链接 (Junction) |
|------|------|------|
| 软件能识别吗 | ❌ 软件会当成 .lnk 文件 | ✅ 软件以为就是真实文件夹 |
| 命令行能用吗 | ❌ `cd` 进去是 .lnk 本身 | ✅ `cd` 直接进入 D 盘真实目录 |
| 能"骗过"安装程序吗 | ❌ 不行 | ✅ 完美欺骗 |

### Q2：创建后 C 盘能释放多少空间？

C 盘上只留下一个不到 1KB 的"指路牌"。原来占用的空间 100% 转移到 D 盘。

### Q3：符号链接会影响软件更新吗？

用 `/J` 创建的目录联接对软件**完全透明**，自动更新不受任何影响。这也是为什么 VS Code 用这个方案能完美运行。

### Q4：能跨盘创建符号链接吗？

能。本文的所有例子都是 C 盘 → D 盘的跨盘链接，这正是最常用的场景。
